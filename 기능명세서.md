## 1. 서비스 개요

친구의 시간 약속 준수를 가상의 '주식'으로 표현하고, GPS 기반 도착 인증 결과가 약속 단위로 주가에 반영되는 소셜 게임형 웹 서비스. 유저는 친구의 특정 약속에 대해 정시 도착에 투자(매수 포지션)하거나 지각에 베팅(공매도 포지션)하며 즐긴다.

이 서비스는 지각을 **교정하겠다고 약속하지 않는다.** 아무도 공식적으로 다루지 않던 '시간 약속'이라는 사회적 규칙을, 처벌이 아니라 게임으로 **표면 위에 올리는 것**이 목적이다.

## 3. 사용자 정의

| 구분 | 설명 |
|---|---|
| 유저 | 서비스에 가입한 모든 사용자 |
| ┗ 투자자 역할 | 다른 유저의 특정 약속에 대해 매수/공매도 포지션을 연다 |
| ┗ 종목 역할 | 본인이 발행한 주식의 대상. 수동적 감시 대상이 아니라, 정시 도착으로 자기 주가를 방어하는 능동적 플레이어(F-15). 자기 주식에 대한 투자자 권한은 특례(P-5, 5.5장)를 따른다 |
| ┗ 약속 참여자 | 약속에 초대되어 **수락한** 유저. 수락자만 GPS 인증·판정 대상이 되며, 수락자에 대해서만 해당 약속 베팅이 열린다(F-19) |

모든 유저는 투자자이자 종목이다. 단, 종목이 되는 것은 친구 수락(F-02), 약속의 판정 대상이 되는 것은 초대 수락(F-19)을 각각 전제로 한다.

## 4. 핵심 개념 정의

| 개념 | 정의 |
|---|---|
| **현재가** | 해당 종목의 최근 정산가. 약속 정산 시에만 갱신되며, 약속 사이에는 변하지 않는다 (모델 B) |
| **약속 종료 시각** | 약속 시각 + 노쇼 상한 60분(`NO_SHOW_MINUTES`). 이 시각에 모든 참여자의 판정이 확정 가능해진다 |
| **포지션** | (유저, 종목, 대상 약속, 방향[매수/공매도], 수량, 잠금 포인트)의 묶음. **반드시 특정 약속에 연결**된다 |
| **베팅 가능 구간** | 약속 생성 시점 ~ 약속 시각. 약속 시각 이후에는 인증 현황(미인증=지각 진행 중)이 노출되므로 신규 포지션을 금지한다(확정 정보 베팅 차단) |
| **잠금 포인트** | 포지션 개설 시 수량 × 개설 시점 현재가 만큼 가용 포인트에서 잠금. 정산 시 전액 반환 후 손익 반영 |
| **손익** | 수량 × (해당 약속 정산이 유발한 가격 변동분). 매수 = 수량 × (정산 후가 − 정산 전가), 공매도 = 수량 × (정산 전가 − 정산 후가). **다른 약속의 정산으로 인한 가격 변동은 손익에 포함되지 않는다.** 손실은 잠금 포인트를 상한으로 클램프 |
| **카운터파티** | 시스템(하우스). 손익만큼 시스템이 포인트를 발행/회수한다. 매수자-공매도자 매칭 없음. 랭킹(S-06)은 절대 포인트가 아닌 수익률 기준으로 산정한다 |
| **실보유 주식** | 자기주식(F-17로 취득)만 존재. 타인 주식은 보유 개념이 없다(포지션만 존재) |

### 5.2 약속 및 GPS 인증

| ID | 기능명 | 상세 설명 | 구현 |
|---|---|---|---|
| F-04 | 약속 생성 | 장소(좌표), 약속 시간, 초대 친구 목록 입력. **초대 가능 대상 = 생성자의 친구만(R-6)**. 생성자는 자동으로 수락 상태의 참여자가 된다. 초대된 친구는 F-19 응답 전까지 '미응답' 상태 | ✅ |
| F-19 | 약속 초대 응답 | 초대받은 유저는 약속을 수락하거나 거절한다. **수락한 유저만 판정 대상이자 베팅 대상이 된다.** 거절·미응답 유저는 해당 약속에서 판정·베팅 대상에서 제외(주가 영향 없음). 약속 시각 도래 시점까지 미응답이면 자동 거절 처리 | ⚠️ **API만**(`POST /api/promises/:id/respond`). `PromiseDetailScreen`은 참여자 상태를 지도 오버레이 칩으로 표시만 하고 수락/거절 버튼이 없다 — 실제 작동 경로는 백엔드 자동 거절뿐 |
| F-05 | GPS 도착 인증 | 수락한 참여자는 도착 후 인증 버튼을 눌러 도착을 기록한다. 약속 장소 반경 50m(`CHECKIN_RADIUS_METERS`) 이내에서만 인증. **인증 가능 구간 = 약속 생성 ~ 약속 종료 시각.** 조기 도착 인증은 정시로 처리, 종료 시각 이후 인증 시도는 거부(노쇼 확정). **[공개 규칙 R-1]** 약속 시각 전에는 인증 여부·시각을 본인에게만 표시하고 타 유저에게는 마스킹한다(확정 정보로 무위험 베팅 차단) — 약속 시각(=베팅 마감) 후에는 전체 공개 | ✅ |
| F-06 | 지각 판정 | 인증시각 ≤ 약속시각 → 정시. 초과 시 지각, 분 단위 올림(`Math.ceil`)으로 기록. 종료 시각까지 미인증 → **노쇼 = 지각 60분으로 간주(단리)**. 판정은 정산 배치(F-12)에서 일괄 확정 | ✅ |

### 5.3 주가 시스템

| ID | 기능명 | 상세 설명 | 구현 |
|---|---|---|---|
| F-07 | 주가 변동(약속 단위 정산) | **약속 정산 시 1회만 변동(모델 B 확정).** 정시 → `current + floor(current × 0.03)`(`ON_TIME_RISE_RATE`), 지각/노쇼 → `current − floor(current × lateMinutes × 0.01)`(`LATE_DROP_RATE_PER_MIN`), 하한가 1,000원(`FLOOR_PRICE`)에서 클램프. 랜덤 변동 없음(P-1). 노쇼는 `lateMinutes=60` 고정 적용. 수치 제약 "분당 하락률×60≤100%"는 1%×60=60%≤100%로 충족하지만, 최종 손실 보장은 손실 클램프(4장)가 담당 | ✅ |
| F-08 | 주가 히스토리 저장 | 약속(정산) 회차별 정산가를 시계열로 저장. 별도 히스토리 테이블 없이 `promise_participants.settled_price`가 겸용 | ✅ |

| F-10 | 매수 포지션(정시 베팅) | 유저는 친구의 **특정 약속**에 대해 정시 도착에 베팅한다. **베팅 대상 = 내 친구인 종목만.** 같은 약속의 비친구 참여자는 현황에 이름만 표시되고 베팅 불가(R-5). **대상 약속 선택이 필수 입력**(수락 참여자·베팅 마감 전 약속만). 수량 × 현재가 포인트 잠금. 손익 미리보기 제공(I-1). 자기 주식 불가(P-5, F-17로만). 동일 (유저,종목,약속)에 방향 불문 **포지션 1개 제한** | ✅ |
| F-11 | 공매도 포지션(지각 베팅) | F-10과 대칭 구조(약속 선택 필수, 잠금, 마감, 1포지션 제한 동일). 자기 주식 불가. 손익 = 수량 × (정산 전가 − 정산 후가), 손실은 잠금 한도 | ✅ |
| F-12 | 자동 정산 | 약속 종료 시각에 시스템이 판정·가격 갱신·포지션 정산을 일괄 수행한다. **트리거: 서버 스케줄러가 1분 주기로(`POLL_INTERVAL_MS`=60,000ms) `settle_due_at ≤ now AND settled_at IS NULL` 약속을 `settle_due_at ASC` 순으로 폴링**하여 한 트랜잭션에서 ①미응답 초대 `invited`→`auto_declined` 일괄 처리 → ②참여자별 판정 확정(F-06) → ③종목별 가격 갱신(F-07) + EWMA(L-01)·정시 스트릭(S-01) 갱신 + 해당 종목의 open 옵션 정산(S-04) → ④해당 약속의 전 open 포지션 정산(잠금 반환+손익, F-07 결과 기준) → ⑤정시 확정 참여자에게 방어 보상(F-15) 지급 + F-17 특별매수 권한 발급 → ⑥`promises.settled_at` 기록(멱등 키)으로 처리한다. **멱등 보장**(재실행 무해, 서버 재기동 시 자동 복구). **도달 보장:** 정산 결과는 미확인 상태로 기록되고, 홈·자산 화면의 "미확인 정산 N건" 배너로 투자자·종목 본인 양쪽에 노출 | ✅ |

### 5.9 핵심 기능 수용 기준 (Acceptance Criteria)

> 핵심 루프 "초대 수락 → GPS 인증 → 판정 → 베팅 → 정산"과 자기주식 규칙에 완료 조건을 정의한다. Given–When–Then 형식. **아래 AC는 전부 백엔드 로직 기준이며, 실제로 실행 가능한지는 5.1~5.6장의 "구현" 열을 함께 봐야 한다**(예: F-19의 "When 수락" 분기는 API로는 성립하지만, 프런트에 그 API를 부르는 버튼이 없다).

**F-19 약속 초대 응답**
- Given 초대받은 유저 / When 수락 / Then 참여자(판정·베팅 대상)로 등록 — ⚠️ 이 분기를 프런트에서 수동으로 트리거할 UI가 없음
- Given 초대받은 유저 / When 거절 / Then 판정·베팅 대상 제외, 주가 영향 없음 — ⚠️ 동일
- Given 미응답 상태 / When 약속 시각 도래 / Then 자동 거절 처리 — ✅ 유일하게 실제로 발생하는 경로
- Given 미응답·거절 유저 / When 해당 약속으로 베팅 시도 / Then 거부 (베팅 대상 아님)

**F-05 GPS 도착 인증**
- Given 수락한 참여자 & 반경 50m 이내 & 종료 시각 전 / When 인증 / Then 성공 + (유저, 약속, 인증시각) 저장
- Given 반경 50m 밖 / When 인증 시도 / Then 거부 + "범위 밖" 안내(`distanceMeters` 포함)
- Given 약속 종료 시각 경과 / When 인증 시도 / Then 거부 (노쇼 확정)
- Given 이미 인증한 참여자 / When 재인증 / Then 최초 인증만 유효 (`checkin_at IS NULL` 조건부 UPDATE로 원자적 차단)
- Given 미수락 유저 / When 인증 시도 / Then 거부
- Given 약속 시각 전 & 참여자 A 조기 인증 완료 / When 타 유저가 참여자 현황 조회 / Then A의 인증 여부·시각 비노출(마스킹, R-1) — 본인 조회 시에는 표시
- Given 약속 시각 경과 / When 참여자 현황 조회 / Then 전원 인증 상태 공개

**F-06 지각 판정**
- Given 인증시각 ≤ 약속시각 / Then 정시
- Given 인증시각 > 약속시각 / Then 지각 + 초과 분(올림) 기록, 최대 59분으로 클램프(60분은 노쇼 전용)
- Given 종료 시각까지 미인증 / Then 지각 60분(노쇼 상한, 단리)으로 확정

**F-10 매수 포지션 / F-11 공매도 포지션**
- Given 대상이 타인 종목 & 해당 종목의 수락 참여 약속이 마감 전 & 가용 포인트 ≥ 수량×현재가 & 동일 (종목,약속) 포지션 없음 / When 매수·공매도 실행 / Then 포지션 생성 + 수량×현재가 잠금
- Given 약속 시각 경과(베팅 마감) / When 매수·공매도 시도 / Then 거부
- Given 동일 (종목,약속)에 기존 포지션(방향 불문) 존재 / When 시도 / Then 거부
- Given 대상이 자기 주식 / When 시도 / Then 거부 (F-17로만)
- Given 가용 포인트 부족 / When 시도 / Then 거부
- Given 대상이 비친구 / When 시도 / Then 거부

**F-12 자동 정산**
- Given 종료시각 ≤ 현재시각 & 미정산 약속 / When 스케줄러 실행 / Then 판정 확정 → 가격 갱신 → 전 포지션 정산(잠금 반환+손익) → 보상/권한 발급, 순서 보장
- Given 손실 > 잠금 포인트 / When 정산 / Then 손실을 잠금액으로 클램프 (가용 포인트 음수 불가)
- Given 이미 정산된 약속 / When 재실행 / Then 무시 (멱등)
- Given 정산 완료 / Then 관련 투자자·종목 본인 전원에게 미확인 정산 레코드 생성 → 홈/자산 배너 노출
- Given 미확인 정산 보유 유저 / When 정산 내역 화면 진입 / Then 확인 처리, 배너 소거

**F-17 자기 주식 특별 매수**
- Given 정산에서 본인 정시 확정 / Then 특별매수 권한 발급(한도 3주, 유효 24h) — ✅ 발급 자체는 자동으로 일어남(정산 배치가 DB에 INSERT)
- Given 유효 권한 보유 / When 한도 내 매수 / Then **행사 시점 현재가**로 체결, 보유 자기주식 등록 — ⚠️ "When 한도 내 매수"를 트리거할 UI가 없어 이 분기는 실행되지 않음
- Given 권한 발급 후 타 약속 정산으로 현재가 변동 / When 행사 / Then 발급 시점 가격이 아닌 **행사 시점 현재가** 적용 (즉시 매도 차익 루프 차단, R-2)
- Given 한도 초과 수량 / When 매수 / Then 초과분 거부
- Given 권한 없음(평상시) / When 자기주식 매수 시도 / Then 거부
- Given 발급 후 24시간 경과 & 미행사 / Then 권한 소멸

**F-18 자기 주식 매도**
- Given 보유 자기주식 & 현재가 > 취득가 / When 매도 / Then 현재가 즉시 체결 + 차익 반영 — ⚠️ F-17이 막혀 있어 보유 자기주식 자체가 생기지 않으므로 실질적으로 도달 불가능한 상태
- Given 현재가 ≤ 취득가 / When 매도 시도 / Then 거부 (손절 불허)

| S-03 | ETF(묶음 펀드) | 여러 친구를 묶은 바스켓 주문. **`quantity`·`direction`은 바스켓 전체 공통값**이며 leg별 개별 지정은 불가능하다. 구성 종목(leg)은 2~5개(`ETF_BASKET_MIN_LEGS`/`MAX_LEGS`). 정산은 각 leg가 걸린 약속이 개별적으로 정산될 때 기존 정산 엔진이 그대로 처리(바스켓 전용 정산 로직 없음, 한 바스켓 안에서도 leg별 부분 정산 가능). **추천 테마는 지각 이력(EWMA) 기반 3인 그룹 단위로만 매치**(risky/safe 각 최대 1개, 후보가 3명 미만이면 해당 side는 추천 자체가 생성되지 않음) — 애초엔 1인 그룹 추천 규칙도 있었으나 바스켓 최소 leg 수(2개)와 모순되어 제거됨 | ✅ |
| S-04 | 옵션 거래(콜/풋) | 특정 약속의 지각 여부(strike 없는 이진 상품)에 고위험·고수익 베팅. EWMA 지각확률 기반 프리미엄(L-01, 풋=기준가×p×수량 / 콜=기준가×(1-p)×수량), 콜=정시 승리·풋=지각/노쇼 승리, 패배 시 이미 낸 프리미엄만 손실(상한 고정, 추가 손실 없음). 전용 `option_positions` 테이블 사용(`positions`와 별도) | ✅ |
| S-05 | 레버리지 베팅 | (모델 B 재정의) 포지션 손익에 배수(1x/3x/5x/10x, `ALLOWED_MULTIPLIERS`) 적용. 잠금(마진)은 배율과 무관하게 고정 — 배율이 클수록 같은 지각분에도 손실이 잠금 한도에 더 빨리 도달한다. 실시간 마진콜 없음 — **정산 시 1회 판정**하며 손실은 잠금 전액 한도로 청산. 청산 조건은 개설 시점에 확정 고지(L-02) | ✅ |
| S-06 | 랭킹 / 커뮤니티 | 내 친구 범위 한정. 최고 **수익률**(절대 포인트 아님 — 시스템 카운터파티로 인한 포인트 총량 변동과 무관하게 공정) 랭킹 + 나·친구들의 최근 정산 소식을 모은 활동 피드(`GET /api/friends/activity-feed`) | ✅ (`RankingCard`·`FriendActivityFeed`가 별도 화면 없이 `/home`에 인라인 노출 — v7의 SC-21 전용 화면은 없음) |
| S-07 | 알림 / 푸시 | 정산 완료·친구 요청·약속 초대 등을 인앱 알림함으로 통합(`GET /api/me/notifications`, 1차 미확인 배너 F-12의 고도화). 실시간 push 인프라는 스코프 아웃(8.1장) | ✅ (인앱 알림함까지만. 친구요청/약속초대 알림 항목을 눌러도 각 목록 화면으로 이동할 뿐이라 F-02/F-19의 UI 공백이 여기서도 그대로 드러남) |
| S-08 | 정산 전 포지션 취소 (재정의) | v5의 "임의 시점 숏 청산"은 모델 B에서 약속 사이 가격이 정지하므로 무의미 → **베팅 마감(약속 시각) 전 포지션을 현재가 기준으로 즉시 정산(조기 청산)**하는 것으로 대체(`POST /api/positions/:id/close`). 취소 수수료 없음(전액 반환) | ✅ (`OpenPositionsPanel`) |
| S-09 | 이모지 리액션 | 종목 상세·정산 결과에 정해진 이모지(😱📉🛡️🔥, `ALLOWED_REACTIONS`)로 반응. 자유 텍스트 없음(P-4 가드레일을 구조로 보장). 재반응 시 갱신 | ✅ (`ReactionBar`, 정산 결과 화면) |

### 6.1 파생상품 프리미엄·청산 로직 (규칙 기반, 담당: S-04·S-05)

> AI/ML·LLM 호출 없이 통계 공식(EWMA)과 임계값 트리거 + 문자열 템플릿으로 구현한다.

| ID | 기능명 | 상세 설명 |
|---|---|---|
| L-01 | 동적 프리미엄 산정 (S-04 연계) | 친구별 지각 이력에 EWMA 적용: `p_t = α × I_late + (1-α) × p_(t-1)`, **α = 0.25, 초기값 p₀ = 0.5**(둘 다 확정값, `EWMA_ALPHA`/`EWMA_P0`). `풋 프리미엄 = 기준가 × p × 수량`, `콜 프리미엄 = 기준가 × (1-p) × 수량`. 약속 정산(F-12) 시점에 재계산·저장하는 배치 로직. 산출된 p는 종목 상세에 "지각 위험도 N%" 배지로도 노출(I-5, `StockRankingTable`/`OptionOrderPanel`) |
| L-02 | 청산 조건 고지 (S-05 연계, 모델 B 재정의) | v5의 실시간 마진콜은 모델 B에서 성립 불가하여 폐기. 레버리지 포지션 **개설 시점에 청산 조건을 결정론적으로 계산해 고지** — 템플릿: `이번 약속에서 ${friendName}이(가) ${thresholdMinutes}분 이상 지각하면 잠금 포인트 전액이 청산됩니다.`(매수) / 정시 도착 시 전액 청산(공매도, 단발 이벤트라 임계값이 아닌 참/거짓 판정). 정산 시 조건 충족 여부 1회 판정. `OrderPanel`에서 배율>1 선택 시 노출 |

## 10. DB 스키마 요약

> 전체 DDL은 `db/schema.sql`(PostgreSQL 16), ERD 다이어그램은 `README.md`의 "DB 스키마 → ERD" 절(Mermaid) 참고.

**테이블(11개):** `users`(계정+종목+지갑 병합, 현재가·EWMA·연속 정시 포함) · `friendships`(F-02) · `promises`(F-04) · `promise_participants`(F-05/F-06/F-19, 가격 히스토리 겸용) · `positions`(F-10/F-11, 레버리지 배수 포함) · `etf_orders`(S-03, 바스켓 헤더) · `option_positions`(S-04 — **v7 문서엔 없었고, 실제로도 `db/schema.sql`에서 통째로 누락돼 있던 것을 이번에 발견·추가**) · `self_stock_options`/`self_stock_lots`(F-17/F-18) · `point_transactions`(F-09/F-14/F-15 원장) · `reactions`(S-09).

**DB 레벨로 보장되는 핵심 불변식(CHECK/UNIQUE):**
- 정산은 약속당 1회만(`settled_at` 유무 기반 멱등)
- 자기 자신 종목에 대한 포지션 생성 차단(`positions.investor_id <> stock_user_id`) — **ETF leg·옵션(`option_positions`)에도 전용 제약 없이 동일하게 적용**
- 잠금 포인트 = 수량 × 개설 시점 현재가(`locked_points = quantity * open_price`), 손실은 잠금 포인트로 클램프(`payout >= -locked_points`)
- 동일 (유저,종목,약속) 조합에 방향 불문 포지션 1개 제한(`uq_position_once`)
- 판정(`verdict`)-지각분(`late_minutes`)-정산가 정합(정시=0분 / 지각=1~59분 / 노쇼=60분 고정)
- 자기주식 손절 불허(`sold_price > acquired_price`)
- 옵션 배당은 항상 0 이상(`option_positions.payout >= 0`) — 손실은 이미 지불한 프리미엄으로 상한 고정
- `positions.etf_order_id`는 `etf_orders(id)` FK — 존재하지 않는 바스켓 참조 차단
